### Tuesday

I'm back. That's it.

What am I up to right now you ask ? I'm making a game engine. *What's that you ask*, it's the engine used to make a game. 

Sarcasm aside, I joined this [server](discord.gg/programming) that I'd highly recommend you join as well, it's a Discord server that has some of the smartest, funniest and nicest people I've had the pleasure of talking to, and one of them suggested that we make a game engine together. 

A game engine is something that you use to make the actual game with, our game engine is not based on Pixels but [Voxels](https://en.wikipedia.org/wiki/Voxel), a Voxel as I describe to other people right now is basically a 3D Pixel with metadata, and because of that you can do a lot of cool stuff with physics because you can define the action of every particular voxel to some event and it looks pretty [cool](https://www.youtube.com/watch?v=hSSDp5f6L7k). We planned on writing it in **Vulkan** but hey, we both don't know *shid* about Graphics and it takes close to a 1000, yes you read it right **1000** lines of code to draw a **TRIANGLE** on the screen with. So we decided on OpenGl, we were going to make it Cross-Platform, use [Vagrant](https://vagrantup.com), to keep the build Environments same and use CMake, but we ran into problems, (could'nt expose the graphics card in the Vagrant Box and CMake is complicated), so right now the project is just set up for Visual Studio, but I plan on using PreMake to automate that process. The Goal right now is just to get through the [OpenGl-Tutorial](http://www.opengl-tutorial.org) to get a basic understanding of how to render stuff on the screen and then get going with making the engine. 

The reason I'm doing this right now is because if not for this I would still be on GitHub trying to find an organization to contribute to, make a list out of them, take days to select one, don't like the Org and repeat the process over and over and not be productive. This will teach me a lot of stuff, a lot of disciplines of computer science and i'll finally understand how people go from **c=a+b** in C++ to **Doom**.

Also I've had to pick up Competitive-Coding, I don't like doing the questions, but it's important to get a job, **a necessary evil**, so i'll also be posting any questions I did from now on. 

Also I've gotten into reading again, I just completed reading about the exploits of Arthur Dent in the the Book **The Hitchhiker's Guide to the Galaxy** by **Douglas Adams** and plan to set up the books page so that I can talk about the book, so look forward to it.
